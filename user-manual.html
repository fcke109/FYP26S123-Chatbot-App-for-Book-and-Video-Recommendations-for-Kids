1. Introduction

1.1 Document Purpose

This document serves as the official user manual for the Little Dino application. It provides detailed instructions on how to install, configure, and use all features of the platform. This manual is intended to guide developers through the setup process and to help end users understand the functionality available to them.

1.2 Target Audience

This manual is intended for the following audiences:

  • Developers who need to set up the development environment, deploy the backend, and build the Android application.
  • Children (ages 3-12) who will use the chatbot to discover books and videos.
  • Parents who wish to configure parental controls and monitor their child's content access.
  • Administrators who manage the content library and user accounts on the platform.

1.3 Scope and Purpose

Little Dino is an AI-powered Android chatbot application that provides personalised book and video recommendations for children aged 3 to 12. The application uses OpenAI's GPT models to deliver conversational recommendations and a custom ANN-inspired recommendation engine to rank content based on each child's age, interests, reading level, and favourite history.

The purpose of the application is to encourage reading and learning among children by providing a safe, engaging, and personalised content discovery experience. Parents can configure safety settings through PIN-protected parental controls, while administrators manage the content library through a dedicated dashboard with AI-assisted book curation tools.

2. Installation Instructions

2.1 Prerequisites

Before setting up the application, ensure the following software and accounts are available.

Software Requirements:

  • Android Studio (latest stable version, Ladybug or newer)
  • Java Development Kit (JDK) 11 or higher
  • Node.js version 18 and npm (Node Package Manager)
  • Firebase CLI (install by running: npm install -g firebase-tools)
  • Git version control

Account Requirements:

  • An OpenAI API key obtained from platform.openai.com
  • A Firebase project with Authentication, Cloud Firestore, Cloud Functions, and Cloud Storage enabled
  • An Android device running Android 8.0 (API 26) or higher, or an Android emulator configured in Android Studio

2.2 Backend Set Up

Step 1: Clone the repository from GitHub to your local machine.

Step 2: Open a terminal and navigate to the firebase/functions/ directory within the project.

Step 3: Install the Node.js dependencies by running: npm install

Step 4: Log in to Firebase by running: firebase login

Step 5: Configure the OpenAI API key for Cloud Functions by running: firebase functions:config:set openai.key="YOUR_OPENAI_API_KEY"

Step 6: Deploy the backend services by running: firebase deploy --only functions,firestore:rules,storage

Step 7 (Optional): To run the backend locally for testing, run: npm run serve. This starts the Firebase emulator suite with Authentication on port 9099, Cloud Functions on port 5001, Firestore on port 8080, Storage on port 9199, and the emulator dashboard on port 4000.

2.3 Frontend Set Up

Step 1: Open Android Studio and select "Open an Existing Project". Navigate to the root directory of the cloned repository and open it.

Step 2: Create a file named local.properties in the project root directory. Add the following line, replacing the placeholder with your actual OpenAI API key: OPENAI_API_KEY=your_openai_api_key_here

Step 3: Download the google-services.json file from the Firebase Console under Project Settings and place it in the app/ directory.

Step 4: Wait for Android Studio to sync the Gradle files automatically. If prompted, click "Sync Now" to resolve all dependencies.

Step 5: Connect a physical Android device with USB debugging enabled (running Android 8.0 or higher) or launch an Android emulator from the AVD Manager.

Step 6: Click the Run button (green play icon) in Android Studio or select Build, then Run from the menu bar. The application will compile and install on the connected device or emulator.

3. User Roles

The application supports three distinct user roles, each with different access levels and capabilities.

3.1 Child User

The child user is the primary user of the application. Children can chat with the Little Dino AI chatbot to receive personalised book and video recommendations. They can browse the curated book library, save books and videos to their favourites collection, and read or watch content through the safe web viewer. Children can also customise their profile by selecting interests and setting their reading level to improve recommendation accuracy.

3.2 Parent User

The parent user accesses the platform through the same account as their child. Parents can access PIN-protected parental controls from the Profile screen to configure safety settings. These settings include setting a maximum age rating for content, toggling video content on or off, and managing content filters. The parental controls ensure that children only see age-appropriate material.

3.3 Admin User

The admin user has full access to the platform's management features. Admins are routed to a dedicated admin dashboard upon login instead of the standard chat interface. From the dashboard, admins can view and manage all registered users, search for books across multiple external APIs, add or remove books from the curated library, and use the AI-powered book curator to automatically extract book metadata from text. The admin account is identified by the email address admin@littledino.com with the plan type set to ADMIN.

5. Registration and Login

5.1 Creating an Account

New users can create an account from the Registration screen. The registration process requires the following information:

  • Name: The user's display name.
  • Email: A valid email address used for login.
  • Password: A password with a minimum of 6 characters.
  • Age: The child's age (numeric value).
  • Interests: At least one interest must be selected from the following options: Reading, Science, Animals, Adventure, Fantasy, Art, Music, Sports, History, and Nature.
  • Reading Level: One of four levels must be selected: Beginner, Early Reader, Intermediate, or Advanced.

Upon successful registration, the user is automatically logged in and directed to the Chat screen. The account is created with a FREE plan type by default. All user data is stored securely in Firebase Authentication and Cloud Firestore.

5.2 Logging In

Existing users can log in from the Login screen by entering their registered email address and password. Upon successful authentication, the application checks the user's role:

  • If the user's email is admin@littledino.com or their plan type is ADMIN, they are directed to the Admin Dashboard.
  • All other users are directed to the main application with the Chat screen as the default view.

5.3 Logging Out

Users can log out from the Profile screen (for regular users) or the Admin Dashboard (for admin users) by tapping the Logout button. Logging out clears the authentication session and returns the user to the Login screen.

6. Chat Screen

The Chat screen is the primary interface of the application where children interact with Little Dino, the AI chatbot. The screen displays a conversation view with message bubbles and recommendation cards.

6.1 Sending Messages

Users type their message in the text input field at the bottom of the screen and tap the Send button. Messages can include questions about books, requests for recommendations, or general conversation about reading topics. The chatbot processes each message using OpenAI's GPT-4o-mini model and responds with personalised suggestions based on the user's profile.

6.2 Receiving Recommendations

When the chatbot identifies relevant content, it returns recommendation cards alongside its text response. Each recommendation card displays:

  • The book or video title
  • A brief description of the content
  • A personalised reason explaining why this item was recommended
  • A relevance score (0.0 to 1.0) indicating how well the item matches the user's profile
  • An "Add to Favourites" button to save the item
  • An "Open" button to view the content

Recommendations are ranked by the ANN-inspired recommendation engine, which considers the user's interests, age, reading level, and favourite history. The chatbot maintains the last 10 messages of conversation history to provide contextually relevant responses.

6.3 Opening Recommendations

Tapping the "Open" button on a recommendation card navigates the user to the Safe Web Viewer, where the book or video is displayed within the app. Books are opened through approved reader platforms (StoryWeaver, Archive.org, Open Library), while videos are opened through YouTube.

7. Library Screen

The Library screen provides a visual catalogue of all curated books available on the platform. It is accessible from the bottom navigation bar.

7.1 Browsing Books

Books are displayed in a grid layout showing cover images, titles, and age ranges. Users can scroll through the collection to discover books. The library is populated by the admin through the Admin Dashboard and stored in the gutenberg_books collection in Cloud Firestore.

7.2 Opening a Book

Tapping on a book card opens it in the Safe Web Viewer. Users can also tap the favourite icon on each card to add the book to their favourites collection for quick access later.

8. Favourites Screen

The Favourites screen displays all books and videos that the user has saved to their collection. It is accessible from the bottom navigation bar.

8.1 Viewing Favourites

Favourited items are displayed in a grid layout showing the cover image, title, and description. Users can tap on any item to open it in the Safe Web Viewer. Favourites are stored in Cloud Firestore under the path favourites/{userId}/items/{itemId} and are synchronised in real-time across sessions.

8.2 Removing Favourites

Users can remove items from their favourites by tapping the remove or unfavourite button on the card. The item is immediately deleted from Firestore and disappears from the screen.

9. Profile Screen

The Profile screen allows users to view and edit their personal information and access parental controls. It is accessible from the bottom navigation bar.

9.1 Editing Profile

Users can update the following profile fields:

  • Name: Change the display name shown in the application.
  • Age: Update the user's age, which affects recommendation scoring.
  • Interests: Select or deselect interests from the available options (Reading, Science, Animals, Adventure, Fantasy, Art, Music, Sports, History, Nature). Changes to interests immediately affect future chatbot recommendations.
  • Reading Level: Change between Beginner, Early Reader, Intermediate, and Advanced. This affects how the recommendation engine scores books based on difficulty.

All profile changes are saved to Cloud Firestore and take effect immediately for subsequent chatbot interactions.

9.2 Parental Controls

The Parental Controls section is accessible from the Profile screen and is protected by a 4-digit PIN. After entering a valid PIN, parents can configure the following settings:

  • Maximum Age Rating: A slider control ranging from 3 to 18 that sets the upper limit for content age ratings. Books and videos with age ratings above this threshold will be filtered from recommendations.
  • Allow Videos: A toggle switch that enables or disables video content. When disabled, the chatbot will only recommend books.

These settings are stored in the user's contentFilters object in Cloud Firestore.

10. Admin Dashboard

The Admin Dashboard is the management interface for platform administrators. It is accessible only to users with the ADMIN plan type and replaces the standard bottom navigation with a drawer menu containing four tabs: User Management, Book Library, Add Books, and Settings.

10.1 User Management

The User Management tab displays a list of all registered users on the platform. Each user entry shows:

  • User name and email address
  • Age and selected interests
  • Reading level
  • Plan type badge (displayed with colour coding: Grey for FREE, Gold for PREMIUM, Purple for ADMIN)

The user list is loaded in real-time from the users collection in Cloud Firestore.

10.2 Book Library Management

The Book Library tab displays all books currently in the curated library stored in the gutenberg_books collection. Each book entry shows the cover image, title, author, and age range. Administrators can delete books from the library by tapping the delete button on each entry. Deleted books are permanently removed from Cloud Firestore.

10.3 Adding Books

The Add Books tab provides a search interface for discovering new books to add to the library. When the admin enters a search query, the application performs parallel searches across two external APIs:

  • StoryWeaver: Returns children's picture books with embedded reader URLs.
  • Open Library: Returns books with metadata including Internet Archive identifiers for the embedded reader.

Search results display the book title, author, and cover image. Tapping "Add to Library" saves the book to the gutenberg_books collection in Cloud Firestore, making it immediately available to all users.

10.4 AI Book Curator

The AI Book Curator (accessible through the Curator screen) allows administrators to add multiple books efficiently by pasting raw text. The administrator pastes a block of text containing book information (such as a booklist from a website or document), and the application sends this text to OpenAI's GPT model. The AI automatically extracts structured book metadata including title, author, description, age range, cover image URL, and reader URL. Each extracted book is then saved to Cloud Firestore as a new entry in the curated library.

11. Safe Web Viewer

The Safe Web Viewer (SafeWebViewScreen) provides a protected browsing environment for reading books and watching videos within the application. It ensures that children can only access content from approved domains.

The viewer enforces a domain whitelist that restricts navigation to the following approved sources:

  • StoryWeaver (storyweaver.org.in) for interactive picture books
  • Internet Archive (archive.org) for embedded book readers
  • Open Library (openlibrary.org) for book previews
  • YouTube (youtube.com) for educational videos

The viewer includes the following controls:

  • A back button to return to the previous screen
  • A refresh button to reload the current page
  • A loading indicator that displays while content is being fetched
  • A lock icon indicating that the user is browsing within the safe environment

Any attempt to navigate to a domain not on the whitelist is automatically blocked, ensuring that children remain within safe and approved content at all times.

12.3 Database

The application uses Cloud Firestore, a NoSQL document database provided by Firebase. Firestore stores data in a hierarchical collection-document structure that supports real-time listeners for live data synchronisation.

The Firestore collections are structured as follows:

  • Firebase Storage: Stores book cover images and media assets (storage bucket: fyp-chatbot-81a50.firebasestorage.app).
  • DataStore Preferences (version 1.1.1): Stores local device-level preferences on the Android device.

12.4 Machine Learning Libraries

The application uses two AI components for intelligent recommendations.

OpenAI GPT Models

The chatbot is powered by OpenAI's GPT language models. The Android application calls the GPT-4o-mini model for chat completions and book curation, while the Firebase Cloud Functions use GPT-3.5-turbo for backend chat processing. Both models use a temperature setting of 0.7 to balance creativity with consistency and a maximum token limit of 500 to keep responses concise and child-friendly.

The chat system constructs a personalised system prompt that includes the user's name, age, interests, and reading level. This prompt instructs the model to act as "Little Dino", a friendly dinosaur assistant that recommends age-appropriate books and videos. The model returns structured recommendation data in a JSON block embedded within its response, which the application parses to generate recommendation cards.

ANN-Inspired Recommendation Engine

The custom recommendation engine (RecommendationEngine.kt) is a simplified Artificial Neural Network-inspired scoring system built in Kotlin. It ranks content based on user profile features using a weighted scoring model with four input features:

Interest Match (weight: 0.35): Compares the user's selected interests against the book's title, description, and author name using keyword matching. The score is calculated as the ratio of matching interests to total interests, normalised between 0.0 and 1.0. If the user has no interests set, a default score of 0.5 is assigned.

Age Match (weight: 0.25): Calculates the absolute difference between the user's age and the book's target age range midpoint, computed as (ageMin + ageMax) / 2. The scoring is tiered: a difference of 1 year or less returns 1.0, up to 2 years returns 0.8, up to 3 years returns 0.5, up to 5 years returns 0.3, and larger differences return 0.1.

Reading Level Match (weight: 0.15): Maps both the user's reading level (Beginner = 1, Early Reader = 2, Intermediate = 3, Advanced = 4) and the book's difficulty rating (easy = 1, medium = 2, hard = 3) to numeric values. The score is based on the difference: an exact match returns 1.0, a difference of 1 returns 0.7, a difference of 2 returns 0.3, and larger differences return 0.1.

Favourite Similarity (weight: 0.25): Uses Jaccard similarity to compare a book's text content against the user's previously favourited items. The algorithm tokenises the book's title and description into words (filtering words shorter than 3 characters), then computes the Jaccard coefficient (intersection divided by union) against each favourite. The highest similarity score across all favourites is used. If the user has no favourites, a default score of 0.5 is assigned.

The final relevance score is calculated as the weighted sum of all four feature scores:

Score = (0.35 x Interest) + (0.25 x Age) + (0.15 x ReadingLevel) + (0.25 x FavouriteSimilarity)

The score is clamped between 0.0 and 1.0. Recommendations are sorted by this score in descending order. When the AI service is unavailable, the engine falls back to scoring curated books directly against the user profile and returns the top 3 results.

12.6 Hosting

The application is hosted entirely on Firebase, which runs on Google Cloud Platform infrastructure. The Firebase project (project ID: fyp-chatbot-81a50) provides the following hosted services:

  • Firebase Authentication for user identity management and secure login.
  • Cloud Firestore for real-time NoSQL database storage.
  • Firebase Cloud Functions for serverless backend logic execution.
  • Firebase Storage for media file hosting (storage bucket: fyp-chatbot-81a50.firebasestorage.app).

12.7 Server

The application uses a serverless architecture through Firebase Cloud Functions, meaning there is no dedicated server to manage. Cloud Functions run on Node.js 18 and automatically scale up or down based on incoming request volume. This eliminates the need for server provisioning, maintenance, or capacity planning.

12.8 Deployment

The deployment process consists of two components: the Android application and the Firebase backend.

Android Application Deployment:

The Android application is built using Gradle 9.3.1 with the Android Gradle Plugin 9.0.0. The build process compiles Kotlin source code into an APK (Android Package) or AAB (Android App Bundle) for distribution. The OpenAI API key is stored in a local.properties file that is excluded from version control and injected into the application at build time through the BuildConfig class. Dependency injection is managed by Dagger Hilt 2.59.2, and annotation processing uses KAPT (Kotlin Annotation Processing Tool) with KSP (Kotlin Symbol Processing) 2.0.21-1.0.25 available as an alternative.

Firebase Backend Deployment:

Firebase Cloud Functions are deployed using the Firebase CLI. The deployment process includes an automatic pre-deploy step that compiles TypeScript source files to JavaScript before uploading. Functions are deployed with the command: firebase deploy --only functions. Firestore security rules and Storage rules are deployed separately using: firebase deploy --only firestore:rules,storage. The entire backend can be deployed in a single command: firebase deploy --only functions,firestore:rules,storage.
