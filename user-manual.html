1. Introduction

1.1 Document Purpose

This document serves as the official user manual for the Little Dino application. It provides detailed instructions on how to install, configure, and use all features of the platform. This manual is intended to guide developers through the setup process and to help end users understand the functionality available to them.

1.2 Target Audience

This manual is intended for the following audiences:

  • Developers who need to set up the development environment, deploy the backend, and build the Android application.
  • Children (ages 3–12) who will use the chatbot to discover books and videos.
  • Administrators who manage the content library and user accounts on the platform.

1.3 Scope and Purpose

Little Dino is an AI-powered Android chatbot application that provides personalised book and video recommendations for children aged 3 to 12. The application uses OpenAI's GPT models to deliver conversational recommendations and a custom ANN-inspired recommendation engine to rank content based on each child's age, interests, reading level, and favourite history.

The purpose of the application is to encourage reading and learning among children by providing a safe, engaging, and personalised content discovery experience. Books and videos are accessed through restricted web viewers that only allow navigation to approved domains, ensuring children remain within safe content at all times. Administrators manage the content library through a dedicated dashboard.

2. Installation Instructions

2.1 Prerequisites

Before setting up the application, ensure the following software and accounts are available.

Software Requirements:

  • Android Studio (latest stable version, Ladybug or newer)
  • Java Development Kit (JDK) 11 or higher
  • Node.js version 18 and npm (Node Package Manager)
  • Firebase CLI (install by running: npm install -g firebase-tools)
  • Git version control

Account Requirements:

  • An OpenAI API key obtained from platform.openai.com
  • A Firebase project with Authentication, Cloud Firestore, Cloud Functions, and Cloud Storage enabled
  • An Android device running Android 8.0 (API 26) or higher, or an Android emulator configured in Android Studio

2.2 Backend Set Up

Step 1: Clone the repository from GitHub to your local machine.

Step 2: Open a terminal and navigate to the firebase/functions/ directory within the project.

Step 3: Install the Node.js dependencies by running: npm install

Step 4: Log in to Firebase by running: firebase login

Step 5: Configure the OpenAI API key for Cloud Functions by running: firebase functions:config:set openai.key="YOUR_OPENAI_API_KEY"

Step 6: Deploy the backend services by running: firebase deploy --only functions,firestore:rules,storage

Step 7 (Optional): To run the backend locally for testing, run: npm run serve. This starts the Firebase emulator suite with Authentication on port 9099, Cloud Functions on port 5001, Firestore on port 8080, Storage on port 9199, and the emulator dashboard on port 4000.

2.3 Frontend Set Up

Step 1: Open Android Studio and select "Open an Existing Project". Navigate to the root directory of the cloned repository and open it.

Step 2: Create a file named local.properties in the project root directory. Add the following line, replacing the placeholder with your actual OpenAI API key: OPENAI_API_KEY=your_openai_api_key_here

Step 3: Download the google-services.json file from the Firebase Console under Project Settings and place it in the app/ directory.

Step 4: Wait for Android Studio to sync the Gradle files automatically. If prompted, click "Sync Now" to resolve all dependencies.

Step 5: Connect a physical Android device with USB debugging enabled (running Android 8.0 or higher) or launch an Android emulator from the AVD Manager.

Step 6: Click the Run button (green play icon) in Android Studio or select Build, then Run from the menu bar. The application will compile and install on the connected device or emulator.

3. User Roles

The application supports two distinct user roles, each with different access levels and capabilities.

3.1 Child User

The child user is the primary user of the application. Children can chat with the Little Dino AI chatbot to receive personalised book and video recommendations. They can browse the curated book library, view personalised top picks, save books and videos to their favourites collection, and read or watch content through safe web viewers. Children can also customise their profile by selecting interests and setting their reading level to improve recommendation accuracy. The Profile screen displays a reading history showing recently opened books and videos.

3.2 Admin User

The admin user has full access to the platform's management features. Admins are routed to a dedicated admin dashboard upon login instead of the standard chat interface. From the dashboard, admins can view and manage all registered users, search for books from the Open Library API, add or remove books from the curated library, and seed the library with a starter collection of classic children's books. The admin account is identified by the email address admin@littledino.com with the plan type set to ADMIN.

5. Registration and Login

5.1 Creating an Account

New users can create an account from the Registration screen. The registration process requires the following information:

  • Name: The user's display name.
  • Email: A valid email address used for login.
  • Password: A password with a minimum of 6 characters.
  • Age: The child's age (numeric value).
  • Interests: At least one interest must be selected from the following options: Reading, Science, Animals, Adventure, Fantasy, Art, Music, Sports, History, and Nature.
  • Reading Level: One of four levels must be selected: Beginner, Early Reader, Intermediate, or Advanced.

Upon successful registration, the user is automatically logged in and directed to the Chat screen. The account is created with a FREE plan type by default. All user data is stored securely in Firebase Authentication and Cloud Firestore.

5.2 Logging In

Existing users can log in from the Login screen by entering their registered email address and password. The Login screen displays the Little Dino mascot alongside the title "Little Dino" and the tagline "Your Friendly AI Book & Video Guide!" Upon successful authentication, the application checks the user's role:

  • If the user's email is admin@littledino.com or their plan type is ADMIN, they are directed to the Admin Dashboard.
  • All other users are directed to the main application with the Chat screen as the default view.

5.3 Login Error Messages

When login fails, the application displays user-friendly error messages instead of raw technical errors. The following error messages are supported:

5.4 Logging Out

Users can log out from the Profile screen (for regular users) or the Admin Dashboard (for admin users) by tapping the Logout button. Logging out clears the authentication session and returns the user to the Login screen.

6. Chat Screen

The Chat screen is the primary interface of the application where children interact with Little Dino, the AI chatbot. The screen displays a conversation view with message bubbles and recommendation cards. The header bar shows the Little Dino avatar, the title "Little Dino", the subtitle "Your visual story-time buddy", and a history icon button.

When no messages have been sent, the screen displays a welcome view with the Little Dino mascot and the greeting "Hi! I'm Little Dino!" along with "Ask me for a story or search the library!"

6.1 Sending Messages

Users type their message in the text input field at the bottom of the screen (placeholder text: "Ask Little Dino for a story...") and tap the Send button. Messages can include questions about books, requests for recommendations, or general conversation about reading topics. While the chatbot is processing a response, a typing indicator displays the text "Dino is thinking..." The chatbot processes each message using OpenAI's GPT-4o-mini model and responds with personalised suggestions based on the user's profile.

6.2 Receiving Recommendations

When the chatbot identifies relevant content, it returns recommendation cards in a horizontal scrollable row alongside its text response. Each recommendation card displays:

  • A cover image or placeholder icon (book icon for books, play icon for videos)
  • A type badge in the top-left corner ("BOOK" in blue or "VIDEO" in red)
  • A relevance score badge in the top-right corner showing the ANN match percentage (e.g. "72% Match") in green, displayed only when the score is above zero
  • The book or video title
  • A brief description of the content (up to two lines)
  • A personalised reason explaining why this item was recommended (e.g. "Picked for you because you love Animals and Adventure!")
  • A "Read" or "Watch" button to open the content
  • A heart icon button to add the item to favourites

Recommendations are ranked by the ANN-inspired recommendation engine, which considers the user's interests (35% weight), age (25% weight), reading level (15% weight), and favourite similarity (25% weight). The chatbot maintains the last 12 messages of conversation history to provide contextually relevant responses.

6.3 Opening Recommendations

Tapping a recommendation card or its "Read"/"Watch" button navigates the user to the appropriate viewer:

  • Books: Opened in the Book Reader screen. The application first checks the curated library for a direct reader URL. If no match is found, it searches the Open Library API for a readable edition on Internet Archive. If a readable edition exists, the book opens directly in the reader. Otherwise, an Open Library search page filtered to readable ebooks is displayed.
  • Videos: Opened in the Safe Video Player. Videos typically link directly to YouTube watch URLs provided by the AI chatbot.

When a book or video is opened, the application automatically records the item in the user's reading history, which is displayed on the Profile screen.

6.4 Chat History

Tapping the history icon in the chat header opens a bottom sheet displaying past conversations. Each conversation entry shows a preview of the first message sent and the timestamp of the last activity (formatted as "MMM d, h:mm a", e.g. "Mar 5, 2:30 PM"). Only conversations containing at least one message are displayed.

Users can tap any conversation to reload its messages, or tap the "New Chat" button to start a fresh conversation. Starting a new chat creates a new conversation in Firestore and clears the current message display.

7. Library Screen

The Library screen provides a visual catalogue of books available on the platform. It is accessible from the bottom navigation bar via the book icon tab. The screen displays the heading "My Story Library" with the subtitle "Favourite your best stories to read again!"

7.1 Browsing Books

Books are displayed in a two-column grid layout. Each book card shows the cover image, title, author name, and an age rating badge (e.g. "3–12 yrs"). A heart icon overlay in the top-right corner of each card allows users to add or remove the book from their favourites. The filled heart icon indicates the book is already favourited; tapping it removes the favourite. The outlined heart icon indicates the book is not favourited; tapping it adds the favourite.

The library is populated from the admin-curated content_books collection in Cloud Firestore. When the curated collection is empty, the application automatically loads children's picture books from the Open Library API as a fallback, filtering for books that have a readable edition available online. This ensures that users always have content to browse even before the admin has curated books.

7.2 Top Picks for You

When books are available in the library, a "Top Picks for You" section appears above the main grid. This section displays up to four personalised recommendations in a horizontal scrollable carousel. Each top pick card shows:

  • The book cover image or a placeholder icon
  • A relevance score badge (e.g. "85% Match") in green
  • The book title
  • A personalised reason text explaining why the book was selected (e.g. "Picked for you because you love Science and Nature!" or "A great pick for 8-year-olds!")

Top picks are scored and ranked by the ANN-inspired recommendation engine using the user's profile data. Books that the user has already favourited are excluded from the top picks to ensure fresh suggestions. Tapping a top pick card opens the book in the Book Reader.

7.3 Opening a Book

Tapping on a book card opens it in the Book Reader. The application uses the book's stored reader URL or book URL. If neither is available, it falls back to searching Open Library for a readable edition. Users can also tap the favourite icon on each card to add the book to their favourites collection for quick access later.

8. Favourites Screen

The Favourites screen displays all books and videos that the user has saved to their collection. It is accessible from the bottom navigation bar via the heart icon tab.

8.1 Viewing Favourites

Favourited items are displayed in a two-column grid layout. Each favourite card shows the cover image (or a placeholder with a book or play icon), the title, and a "Read" or "Watch" button colour-coded by type (blue for books, red for videos). Users can tap on any card or its action button to open it in the appropriate viewer.

When no favourites have been saved, the screen displays an empty state with the message "No favorites yet!" and the subtitle "Favorite your best stories from the Library or Chat!"

Favourites are stored in Cloud Firestore under the path favorites/{userId}/items/{itemId} and are synchronised in real-time across sessions.

8.2 Removing Favourites

Users can remove items from their favourites by tapping the filled heart icon on the card. The item is immediately deleted from Firestore and disappears from the screen.

9. Profile Screen

The Profile screen allows users to view and edit their personal information and see their reading history. It is accessible from the bottom navigation bar via the person icon tab.

9.1 Viewing Profile

The Profile screen displays the user's information in a card layout:

  • A circular avatar showing the first letter of the user's name
  • The user's name (large, bold text) and age
  • Email address with an email icon
  • Reading level with a book icon (e.g. "Reading Level: Intermediate")
  • Interests displayed as a row of chips (e.g. "Adventure", "Animals", "Science")

The top bar contains an Edit icon button (to enter edit mode) and a Logout icon button (to sign out).

9.2 Editing Profile

Tapping the Edit icon switches the Profile screen to edit mode. Users can update the following fields:

  • Name: Change the display name shown in the application.
  • Age: Update the user's age (numeric input only), which affects recommendation scoring.
  • Interests: Select or deselect interests from the 10 available options displayed as filter chips in a three-column grid. Changes to interests immediately affect future chatbot recommendations and top picks.
  • Reading Level: Change between Beginner, Early Reader, Intermediate, and Advanced using filter chips. This affects how the recommendation engine scores books based on difficulty.

Two buttons appear at the bottom: "Cancel" discards changes and returns to display mode, and "Save" persists changes to Cloud Firestore. All profile changes take effect immediately for subsequent chatbot interactions and library top picks.

9.3 Recently Read

Below the profile information, a "Recently Read" section appears when the user has opened at least one book or video. This section displays a horizontal scrollable carousel of the last five items the user has accessed. Each entry shows:

  • A cover image or placeholder icon (play icon for videos, book icon for books), colour-coded with a light red background for videos and light blue for books
  • The item title (up to two lines)

Reading history is automatically recorded whenever a user opens a book or video from the chat recommendations, library, or favourites. The history is stored in Cloud Firestore under the path readingHistory/{userId}/sessions/ and ordered by the most recent opening time.

10. Admin Dashboard

The Admin Dashboard is the management interface for platform administrators. It is accessible only to users with the ADMIN plan type and replaces the standard bottom navigation with a navigation drawer containing four sections: User Management, Book Library, Add Books, and Settings.

The navigation drawer displays the Little Dino logo, the heading "Admin Dashboard", navigation items for each section, and a red Logout button at the bottom. The drawer is opened by tapping the menu icon in the top bar.

10.1 User Management

The User Management section displays a list of all registered users on the platform under the heading "Active Users". Each user entry shows:

  • User name (bold text)
  • Plan type badge displayed with colour coding: grey for FREE, gold for PREMIUM, and purple for ADMIN
  • Email address
  • Age

The user list is loaded in real-time from the users collection in Cloud Firestore. A loading spinner is displayed while the data is being fetched.

10.2 Book Library Management

The Book Library section displays all books currently in the curated library stored in the content_books collection. The heading shows "Library Stories" followed by the current book count in parentheses. Each book entry is displayed as a card showing:

  • Cover image thumbnail (70dp, rounded corners)
  • Book ID badge (e.g. "001", "002")
  • Title and author
  • Age range badge (e.g. "3–8 yrs")
  • Difficulty badge with colour coding: green for easy, blue for medium, and orange for hard
  • A delete icon button to remove the book from the library

When the library is empty, an icon and the message "No books in library yet" are displayed. Deleted books are permanently removed from Cloud Firestore.

10.3 Adding Books

The Add Books section provides a search interface for discovering new books to add to the library. The heading reads "Discover ICDL Stories" with the subtitle "Search for English picture books with drawings." When the admin enters a search query and taps the Search button, the application searches the Open Library API for matching books.

Search results are displayed as book cards showing the title, author, cover image, and a relevance score percentage badge indicating how well the result matches the search query. Results are filtered to show books with cover images and are sorted by relevance score. Each result includes an Add icon button. Tapping "Add" saves the book to the content_books collection in Cloud Firestore with a sequential ID (e.g. "001", "002"), making it immediately available to all users in the Library screen.

A loading spinner with the text "Searching visual library..." is displayed while results are being fetched.

10.4 Seeding the Library

The Book Library section includes a "Seed 001–010" button that populates the library with a starter collection of five classic children's books. The seeded books include:

  • The Tale of Peter Rabbit by Beatrix Potter
  • Alice's Adventures in Wonderland by Lewis Carroll
  • Cinderella by Charles Perrault
  • The Jungle Book by Rudyard Kipling
  • The Adventures of Pinocchio by Carlo Collodi

Each seeded book is pre-configured with appropriate age ranges, difficulty levels, tags, cover image URLs, and direct Internet Archive reader URLs. This feature ensures that the platform has content available immediately after initial setup without requiring the admin to search and add books individually.

11. Safe Web Viewer

The application provides two restricted web viewers for accessing content: the Book Reader for books and the Safe Video Player for videos. Both viewers enforce domain whitelists that restrict navigation to approved sources, ensuring that children cannot accidentally navigate to unsafe websites.

11.1 Book Reader

The Book Reader (BookReaderScreen) opens when a user taps on a book recommendation, library book, or favourite book. It displays a top bar with the title "Visual Storybook" and a close button. The book content is loaded in a WebView with the following settings enabled: JavaScript, DOM storage, overview mode, and wide viewport.

The reader restricts navigation to the following approved domains:

  • International Children's Digital Library (childrenslibrary.org)
  • Internet Archive (archive.org and ia80, ia60, ia90, ia70 subdomains)
  • Open Library (openlibrary.org and covers.openlibrary.org)
  • StoryWeaver (storyweaver.org.in)

Any attempt to navigate to a domain not on this list is automatically blocked.

11.2 Video Player

The Safe Video Player (SafeWebViewScreen) opens when a user taps on a video recommendation or favourite video. It displays a coloured header bar (red for videos, primary colour for books) containing a back button, a lock icon with the label "Safe Video Player", the content title, a close button, and a linear progress bar showing loading progress.

The video player restricts navigation to the following approved domains:

  • StoryWeaver (storyweaver.org.in) for interactive picture books
  • Internet Archive (archive.org and ia subdomains) for embedded book readers
  • Open Library (openlibrary.org and covers.openlibrary.org) for book content
  • YouTube (youtube.com, youtu.be, youtubekids.com) for educational videos
  • CDN domains (google.com, gstatic.com, cloudflare.com) required for video playback

The viewer includes zoom controls (without on-screen buttons), back navigation within the WebView history, and automatic HTTPS enforcement. A full-screen loading overlay with a spinner and the text "Loading Video..." or "Opening Visual Story..." is displayed while content loads (below 30% progress).

12.4 Machine Learning Libraries

The application uses two AI components for intelligent recommendations.

OpenAI GPT Models

The chatbot is powered by OpenAI's GPT language models. The Android application calls the GPT-4o-mini model for chat completions, while the Firebase Cloud Functions use GPT-3.5-turbo for backend chat processing. Both models use a temperature setting of 0.7 to balance creativity with consistency and a maximum token limit of 500 to keep responses concise and child-friendly.

The chat system constructs a personalised system prompt that instructs the model to act as "Little Dino", a friendly dinosaur assistant that recommends age-appropriate books and videos. The prompt includes a list of curated books available in the library. The model returns structured recommendation data in a JSON block embedded within its response (delimited by [RECOMMENDATIONS] and [/RECOMMENDATIONS] tags), which the application parses to generate recommendation cards. Each recommendation includes a type (BOOK or VIDEO), title, description, reason, and optionally a URL.

ANN-Inspired Recommendation Engine

The custom recommendation engine (RecommendationEngine.kt) is a simplified Artificial Neural Network-inspired scoring system built in Kotlin. It ranks content based on user profile features using a weighted scoring model with four input features:

Interest Match (weight: 0.35): Compares the user's selected interests against the book's title, description, and author name using keyword matching. The score is calculated as the ratio of matching interests to total interests, normalised between 0.0 and 1.0. If the user has no interests set, a default score of 0.5 is assigned.

Age Match (weight: 0.25): Calculates the absolute difference between the user's age and the book's target age range midpoint, computed as (ageMin + ageMax) / 2. The scoring is tiered: a difference of 1 year or less returns 1.0, up to 2 years returns 0.8, up to 3 years returns 0.5, up to 5 years returns 0.3, and larger differences return 0.1.

Reading Level Match (weight: 0.15): Maps both the user's reading level (Beginner = 1, Early Reader = 2, Intermediate = 3, Advanced = 4) and the book's difficulty rating (easy = 1, medium = 2, hard = 3) to numeric values. The score is based on the difference: an exact match returns 1.0, a difference of 1 returns 0.7, a difference of 2 returns 0.3, and larger differences return 0.1.

Favourite Similarity (weight: 0.25): Uses Jaccard similarity to compare a book's text content against the user's previously favourited items. The algorithm tokenises the book's title and description into words (filtering words shorter than 3 characters), then computes the Jaccard coefficient (intersection divided by union) against each favourite. The highest similarity score across all favourites is used. If the user has no favourites, a default score of 0.5 is assigned.

The final relevance score is calculated as the weighted sum of all four feature scores:

Score = (0.35 × Interest) + (0.25 × Age) + (0.15 × ReadingLevel) + (0.25 × FavouriteSimilarity)

The score is clamped between 0.0 and 1.0. Recommendations are sorted by this score in descending order. The engine is used in two contexts: scoring AI chatbot recommendations in real-time during chat, and generating personalised top picks in the Library screen.

12.6 Hosting

The application is hosted entirely on Firebase, which runs on Google Cloud Platform infrastructure. The Firebase project (project ID: fyp-chatbot-81a50) provides the following hosted services:

  • Firebase Authentication for user identity management and secure login.
  • Cloud Firestore for real-time NoSQL database storage.
  • Firebase Cloud Functions for serverless backend logic execution.
  • Firebase Storage for media file hosting (storage bucket: fyp-chatbot-81a50.firebasestorage.app).

12.7 Server

The application uses a serverless architecture through Firebase Cloud Functions, meaning there is no dedicated server to manage. Cloud Functions run on Node.js 18 and automatically scale up or down based on incoming request volume. This eliminates the need for server provisioning, maintenance, or capacity planning.

12.8 Deployment

The deployment process consists of two components: the Android application and the Firebase backend.

Android Application Deployment:

The Android application is built using Gradle 9.3.1 with the Android Gradle Plugin 9.0.0. The build process compiles Kotlin source code into an APK (Android Package) or AAB (Android App Bundle) for distribution. The OpenAI API key is stored in a local.properties file that is excluded from version control and injected into the application at build time through the BuildConfig class. Dependency injection is managed by Dagger Hilt 2.59.2, and annotation processing uses KAPT (Kotlin Annotation Processing Tool) with KSP (Kotlin Symbol Processing) 2.0.21-1.0.25 available as an alternative.

Firebase Backend Deployment:

Firebase Cloud Functions are deployed using the Firebase CLI. The deployment process includes an automatic pre-deploy step that compiles TypeScript source files to JavaScript before uploading. Functions are deployed with the command: firebase deploy --only functions. Firestore security rules and Storage rules are deployed separately using: firebase deploy --only firestore:rules,storage. The entire backend can be deployed in a single command: firebase deploy --only functions,firestore:rules,storage.
